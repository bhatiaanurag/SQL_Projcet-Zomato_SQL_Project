# Zomato SQL Data Analysis Project

## Project Overview
This project presents a structured SQL-based data analysis case study using a simulated Zomato dataset. The primary objective is to derive meaningful business insights related to customer behavior, product performance, and sales trends through the use of SQL queries.

The project demonstrates how relational databases can be effectively utilized to solve real-world business problems by applying data querying, transformation, and aggregation techniques.

---

## Objectives
The key objectives of this project are:

- To analyze customer purchasing behavior  
- To calculate total revenue generated by each customer  
- To identify the most frequently purchased products  
- To measure customer engagement based on visit frequency  
- To evaluate the impact of membership programs on customer activity  

---

## Database Structure

The database is composed of the following tables:

### 1. Users
- `userid`: Unique identifier assigned to each customer  
- `signup_date`: The date on which the user registered  

### 2. GoldUsers_Signup
- `userid`: Unique identifier of the user  
- `gold_signup_date`: The date on which the user upgraded to a gold membership  

### 3. Sales
- `userid`: Unique identifier of the customer  
- `created_date`: Date of transaction  
- `product_id`: Identifier of the purchased product  

### 4. Product
- `product_id`: Unique identifier of the product  
- `product_name`: Name of the product  
- `price`: Price assigned to the product  

---

## SQL Concepts Applied
The project makes use of the following SQL concepts:

- Data retrieval using SELECT statements  
- Data filtering using WHERE clauses  
- Sorting results using ORDER BY  
- Data aggregation using GROUP BY and HAVING  
- Table relationships using INNER JOIN  
- Use of aggregate functions such as SUM and COUNT  
- Subqueries for advanced data extraction  

---

## Business Problems and Analysis

### 1. Total Amount Spent by Each Customer
The total spending of each customer is calculated by joining the Sales and Product tables and applying aggregation on product prices.

### 2. Number of Days Each Customer Visited
Customer engagement is measured by counting the number of distinct transaction dates for each user.

### 3. Most Purchased Product
The most frequently purchased product is identified by counting the occurrences of each product and sorting the results in descending order.

### 4. First Purchase After Membership Upgrade
This analysis identifies the first transaction made by a customer after becoming a gold member by comparing transaction dates with membership start dates.

---

## Key Insights
- Certain products demonstrate significantly higher purchase frequency compared to others  
- Customers exhibit recurring purchasing behavior over time  
- There is variation in spending patterns across different users  
- Membership programs influence customer purchasing decisions and engagement
